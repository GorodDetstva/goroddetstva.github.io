Linux几大主要安装模式整理

1.Debian系列：DPKG（Debian Package Management System)

这种系统的程序后缀为.deb，使用DPKG作为 Debian Linux 家族的基础包管理系统。dpkg的前端工具有常用的APT（高级打包工具），Aptitude 包管理器和新立得包管理工具。dpkg本身是一个底层的工具。上层的工具，如APT，被用于从远程获取软件包以及处理复杂的软件包关系。

属于这个系列的发行版有Debian,Ubuntu/Xubuntu/Lubuntu,POP!OS,LInux Mint,MXLinux,Deepin等。常用命令如下：

apt list --installed #列出所有已经安装的程序

apt-get install /package name/ #安装

apt-get --reinstall install /packagename/ #重新安装

apt-get -f install #修复依赖关系

apt-get source /package name/ #下载程序源代码

apt-get remove /PackageName/ #卸载程序。保留配置文件。

apt-get --purge remove /PackageName/ #上述两者都卸载

apt-get purge /PackageName/ #卸载上述的同时把需要的依赖程序也删除

2.Red Hat系列：RPM（Red Hat Package Management

这种系统的程序后缀为.rpm，使用rpm作为Red Hat Linux家族的基础包管理器系统。rpm的前端工具有常用的yum黄狗更新器和DNF（yum的改版）。本人没有用过红帽系的发行版，因此就不列出啦，感兴趣的小伙伴可以自己搜寻。

3.Arch系列：Pacman

Pacman是Arch系的特色包管理器，常用命令如下：

pacman -S /package_name/ #安装软件

pacman -R /package_name/ #卸载程序。保留配置文件。

pacman -Rs /package_name/ #上述两者都卸载

pacman -Syu #升级系统和同步仓库数据

4.Flatpak

Flatpak是2016年由同名团队开发的一种包管理器，具有红帽公司的背景。要想使用flatpak，您必须在您的发行版里安装flatpak。

在Debian系中打开终端，输入：

sudo apt-get install flatpak

其他系列自行搜索，部分发行版自带flatpak。以下是它的用法：

flatpak update appID #更新应用，后面参数不加代表更新所有应用

flatpak remote-add name location(URL) #添加一个远程仓库

flatpak remote-delete name #删除仓库

flatpak search APPname #在所有库中查询某个程序的名字

flatpak install repository appID #安装指定库里面的程序，自flatpak 1.2以前程序必须为ID名，1.2后输入程序原名会自动搜索库里的匹配程序包并下载安装。此外，您也可以将本地的flatpak安装包拖到终端中“flatpak install”后安装。

flatpak run appID #运行某个程序，程序为ID名

flatpak uninstall appID #卸载某个程序

5.Snap

Snap是由canocical公司（ubuntu的赞助商）开发的包管理系统，自ubuntu 16.04LTS版本开始自带。不过，该举动反响并不太好，据称会破坏您的依赖关系。除了基于ubuntu的系统，您必须在您的OS上安装snap才可以使用它。

6.Appimage

严格来说，它只是一种可执行文件的格式，类似于windows的绿色版。该文件即点即用，非常方便。要想把他加入程序菜单，可使用AppimageLaucher软件

7.其他

OpenSUSE使用zypper管理器，gentoo使用portage管理器，slackware使用slpkg管理器

除此之外，部分软件会要求你运行.sh或.bundle脚本以安装软件。
