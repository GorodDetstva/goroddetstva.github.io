在VMWare虚拟机中安装MacOS指南

有一天博主突发奇想：我能不能装一台苹果系统的机子呢？网上一查，N卡20系之后没有驱动了......全剧终。

不过不要着急，上帝关上了门，焊死了窗户，但总归会留下一个下水道口的:>借助VMWare工具，我们可以尝试在虚拟机中安装黑苹果。不过还是有些许可惜，实体机上已经安装了Windows和Linux,能在实体机上安装MacOS是多少计算机爱好者的梦想阿！os御三家，唯有mac不得齐全:<

要安装虚拟机的Mac，我们必须先下载VMWare（废话）。我们从这个链接下载windows版本的虚拟机软件。

下载安装之后，我们还要下载一个补丁点我下载 版本3系列补丁适用于windows和linux的vmware的11-15版本，最新版3.0.4则适配了16版本（17版本未知）。旧版本2.0系列请自行上网搜索.最后下载Apple High Sierra镜像 密码1t3e

如果你是windows系统，请同时按下 win 和 R 键，输入 services.msc ，将所有Vmware服务全部关掉（所有以vmware打头的服务项），然后打开你解压好的补丁位置，选中win-install.cmd，右键以管理员身份运行，之后重启。

如果你是GNU/linux用户，请切换到你的补丁目录，右键在终端中打开（或者直接用cd命令切换），输入

sudo ./lnx-install.sh

然后就是配置自己的虚拟机，在选择客户机操作系统时选择apple即可。建议内存分配的稍大一些，硬盘最低50GB

配置完虚拟机后，最重要的一步来了：

如果你是Intel CPU 用户，请打开你配置虚拟机的目录，找到一个.vmx的文件，右键用记事本（或者以其他文本编辑器）打开，在 smc.present ="0"后的一行回车，输入：

smc.version = "0"

并在最后添加：

cpuid.1.eax = "00000000000000010000011010100101"

如果你是 AMD CPU用户，请在上述文件的末尾，用同样的方法打开VMX文件，在末尾添加：

smc.version = "0"
cpuid.0.eax = "0000:0000:0000:0000:0000:0000:0000:1011"
cpuid.0.ebx = "0111:0101:0110:1110:0110:0101:0100:0111"
cpuid.0.ecx = "0110:1100:0110:0101:0111:0100:0110:1110"
cpuid.0.edx = "0100:1001:0110:0101:0110:1110:0110:1001"
cpuid.1.eax = "0000:0000:0000:0001:0000:0110:0111:0001"
cpuid.1.ebx = "0000:0010:0000:0001:0000:1000:0000:0000"
cpuid.1.ecx = "1000:0010:1001:1000:0010:0010:0000:0011"
cpuid.1.edx = "0000:0111:1000:1011:1111:1011:1111:1111"
smbios.reflectHost = "TRUE"
hw.model = "MacBookPro14,3"
board-id = "Mac-551B86E5744E2388"
keyboard.vusb.enable = "TRUE"
mouse.vusb.enable = "TRUE"

之后打开虚拟机，一路配置下去即可。不过当您配置磁盘时，您需要先点击顶栏实用工具-磁盘工具，选择你的硬盘并“抹掉”为MACOS格式，名称随意，之后退出磁盘工具并继续安装即可。

最后进入系统，您需要安装它的驱动VMWare tools,在虚拟机软件中的虚拟机选项卡-可移动设备-CD/DVD（SATA），选择vmware安装目录下的darwin.iso,点击确定后桌面上出现Vmwaretools，点击安装。如果出现“系统扩展已被阻止”，则在顶栏中打开苹果logo-系统偏好设置-安全性与隐私，在允许从以下位置下载的应用中允许 VMWare tools即可。

另外，虚拟机的mac是非常卡滴，拿来浏览网页/聊天/看书/编辑文档（推荐LIBRE OFFICE）/测试环境还算是可以的。
